# 48KNOT PC/모바일 웹 구축 계획

## 1) 시안 요약 (screen.png 기준)
- 단일 랜딩 페이지 구조 (Hero → 가치 → 팀 → 히스토리 → 서비스 → 푸터).
- 다크 톤 + 블루 포인트, 글래스/그라디언트 카드 기반.
- 네비게이션은 데스크톱 기준 상단 고정, 모바일은 햄버거 버튼 필요.
- 핵심 요소: 히어로 카피/CTA, 3열 카드 섹션(가치/서비스), 타임라인, 팀 카드 2열.

## 2) PC/모바일 웹 목표
- PC: 1280px 이상 기준에서 정보 밀도와 시각 임팩트 유지.
- 태블릿: 768~1024px 구간 레이아웃 재배치 최적화.
- 모바일: 360~430px 폭에서 가독성/터치성 유지.
- LCP/CLS 개선(배경 이미지 최적화, 폰트 로딩 정책).
- 스크롤 동선에 맞춘 섹션 간격 및 카드 스택 전환.

## 3) 구축 범위
- 단일 페이지(One-page) 반응형 UI.
- 다국어/다크모드 토글은 범위 외(필요 시 2차).
- 연락/문의 CTA는 버튼 및 mailto 링크 우선.

## 4) 정보 구조/섹션 구성
1. Hero
   - 타이틀/서브 카피/CTA
   - 배경 비주얼, 네트워크 라인 SVG
2. 핵심 가치
   - 3개 카드(ALIGN, POTENTIAL, GRATITUDE)
3. 팀 소개
   - 2인 카드(모바일: 세로 스택)
4. 히스토리
   - 타임라인(모바일: 좌측 라인 + 카드 스택)
5. 서비스
   - 3개 카드(모바일: 세로 스택)
6. 푸터
   - 브랜드/문구 + 서비스/연락처

## 5) 반응형 최적화 요구사항
- 헤더
  - PC: 상단 고정 네비 + CTA 유지.
  - 모바일: 햄버거 버튼 → 클릭 시 오버레이 메뉴 또는 드롭다운.
  - 터치 타겟 최소 44px.
- 타이포그래피
  - PC: H1 48~64px, H2 32~40px, 본문 16~18px.
  - 모바일: H1 32~36px, H2 24~28px, 본문 14~16px.
  - 줄바꿈 규칙: 긴 카피는 수동 개행 최소화.
- 레이아웃
  - PC: 3열 카드, 2열 팀 카드 유지.
  - 태블릿/모바일: 3열 카드 → 1열 스택.
  - 태블릿/모바일: 2열 팀 카드 → 1열 스택.
  - 타임라인 좌측 라인 유지, 불릿/배지 크기 축소.
- 미디어
  - 배경 이미지 WebP(PC/모바일 크기 별도 제공), lazy-loading.
  - 아이콘/폰트 preload 최소화.
- 인터랙션
  - 기본 스크롤 애니메이션은 과하지 않게(필요 시 AOS 수준).

## 6) UI 컴포넌트 목록
- 고정 헤더 + 모바일 메뉴
- 히어로 배경(그라디언트 + 이미지 + 라인 SVG)
- 카드(아이콘 + 타이틀 + 설명 + 태그)
- 타임라인 아이템
- CTA 버튼(Primary / Ghost)
- 푸터 링크 리스트

## 7) 기술 스택 제안
- 정적 HTML + TailwindCSS (현재 코드 기반 유지)
- 배포: 정적 호스팅 (Netlify/Vercel/GitHub Pages)
- 이미지 최적화: Sharp 또는 정적 이미지 빌드

## 8) 구현 일정(예시)
- D1: 시안 정리/콘텐츠 확정
- D2: 반응형 레이아웃 구현 (모바일/태블릿)
- D3: 애니메이션/디테일 보정
- D4: QA 및 성능 최적화

## 9) 체크리스트
- PC 네비게이션/CTA 동작
- 모바일 네비게이션 동작
- 360px 화면 기준 줄바꿈/가독성
- PC 기준 텍스트 밀도/여백 확인
- 이미지/폰트 로딩 최적화
- 주요 섹션 간격 통일
- 스크롤 성능 확인

## 10) 추가 확인 필요 사항
- 실제 CTA/문의 동선 (메일/폼/링크)
- 팀/서비스 콘텐츠 확정본
- 도메인 및 배포 플랫폼
